# kunkun 真实翼蜥桌面小宠物方案

> 独立新方案：本方案基于 `realistic-winged-lizard-assistant-color-ref-v2.png` 这版原创真实风翼蜥形象，不沿用石虎方案。  
> 产品场景：kunkun macOS 桌面智能体，常驻菜单栏，通过快捷键唤出 Spotlight 式悬浮面板。  
> 角色方向：真实 3D 质感、青绿色翼蜥、好奇观察者、桌面陪伴型智能体。

---

## 1. 核心定位

### 1.1 一句话定位

**它是一只住在 Mac 桌面边缘的观察型翼蜥小宠物，帮用户看见信息、抓住线索、整理任务。**

### 1.2 产品角色

这个 IP 不是普通吉祥物，而是 kunkun 智能体的“可视化身体”。

用户按下快捷键时，它出现；用户让它看图、找文件、写内容、拆任务时，它通过动作变化告诉用户“我正在观察、搜索、思考、执行、完成”。

它承担三层价值：

- **陪伴价值**：让桌面智能体不再只是冷冰冰的输入框。
- **状态价值**：把智能体内部状态变成用户能一眼看懂的动作。
- **品牌价值**：形成 kunkun 独有的桌面小宠物记忆点。

## 2. IP 角色设定

### 2.1 名称建议

首推名称：**Lumo**

理由：

- 有光感，适合“观察、发现、灵感”的产品语义。
- 发音轻，适合桌面小宠物。
- 不和石虎、Rocky、PukyPuky 混在一起，能作为新方向独立成立。

备选：

- **Kiki**：更轻快，偏可爱陪伴。
- **Mimo**：和 MiMo 眼睛能力有关联，但容易和模型名混淆。
- **Luki**：有 look / lucky 的联想，适合观察型助手。
- **Pilo**：偏小宠物感，软一些。

### 2.2 角色身份

Lumo 是一只从屏幕光斑里孵化出来的翼蜥小宠物。

它喜欢趴在菜单栏、窗口边缘和输入框旁边，观察用户丢给它的线索。它的蓝色大眼睛能看图片、读屏幕、辨认文件；它的翼膜能展开成“注意力雷达”；它的尾巴会根据任务状态发光。

它不是替用户做决定的神秘生物，而是一个机灵的桌面搭档：帮用户把散乱的信息捡起来、排好队、串成答案。

### 2.3 性格

- 好奇：看到新图片、新文件、新问题会立刻凑近。
- 机灵：反应快，动作轻，像随时准备跳出来帮忙。
- 黏人但不打扰：平时安静待命，有事才出现。
- 认真：进入任务状态后动作收敛，眼睛聚焦。
- 小得意：完成任务后会跳一下、翘尾巴。

### 2.4 口头禅

- “我看到了。”
- “这条线索亮了。”
- “等我找找。”
- “我把它们串起来。”
- “抓住这个灵感。”
- “完成，尾巴都亮了。”

## 3. 视觉系统

### 3.1 主形象

以这张图作为视觉主参考：

`/Users/mac/Desktop/kunkun/assets/ip/realistic-winged-lizard-assistant-color-ref-v2.png`

必须保留的形象特征：

- 青绿色主体
- 黄绿色翼膜边缘
- 青蓝色大眼睛
- 奶黄色腹部
- 橙黄色脸颊点缀
- 粉色口腔
- 真实 3D 生物质感
- 半透明薄翼
- 短爪、卷尾、轻盈姿态
- 蓝灰色科技感背景

需要避免：

- 变成扁平卡通
- 变成软萌幼儿风
- 变成暗黑怪兽
- 变成写实爬虫恐怖感
- 色彩变成暗橄榄、薄荷白、低饱和灰绿
- 直接复刻第三方已有 IP 的具体结构

### 3.2 推荐色彩

| 用途 | 色彩方向 | 说明 |
|---|---|---|
| 主体绿 | 高饱和青绿 / 荧光草绿 | 角色主识别 |
| 翼缘黄 | 柠檬黄 / 暖黄绿 | 让角色更亮、更有生命力 |
| 腹部 | 奶油黄 / 浅米黄 | 平衡高饱和绿色 |
| 眼睛 | 青蓝 / 湖蓝 | 表达“观察、视觉、智能” |
| 口腔 | 柔粉色 | 增加亲和力 |
| 背景 | 蓝灰 / 深青灰 | 提供真实 3D 质感和科技感 |
| UI 点缀 | 玻璃白 / 光点黄 / 浅青 | 呼应 macOS 悬浮面板 |

### 3.3 材质

- 皮肤：湿润但不黏腻，带轻微毛孔和渐变。
- 翼膜：半透明，有薄薄透光感。
- 眼睛：高光明显，青蓝虹膜，像摄像头/视觉传感器。
- 爪子：圆润，不尖锐恐怖。
- UI 元素：毛玻璃、柔光、半透明边缘。

## 4. 桌面小宠物产品形态

### 4.1 三种出现层级

#### A. 菜单栏小状态

默认常驻，不占 Dock，不进入用户工作流。

表现：

- 一个简化的 Lumo 眼睛/小脑袋/尾巴图标。
- 状态变化通过颜色和小动效表达。

适合：

- 待命
- 后端运行中
- 正在思考
- 出错
- 完成任务

#### B. 悬浮面板陪伴

用户按快捷键唤出 kunkun 面板时，Lumo 从边缘探头。

表现：

- 初始在输入框左上角或右下角探出半个身体。
- 用户输入时眼睛跟随光标。
- AI 思考时尾巴轻摆、眼睛聚焦。
- 工具调用时进入对应动作。

适合：

- 输入问题
- 看图
- 搜索文件
- 流式回答
- 工具调用反馈

#### C. 桌面自由小宠物

二期功能，可以让 Lumo 以小浮层形式停在屏幕角落。

表现：

- 平时趴在屏幕边缘睡觉。
- 有任务时醒来。
- 用户拖拽图片/文件到它身上时触发分析。
- 用户长时间不操作时进入休眠。

适合：

- 桌面陪伴
- 拖拽入口
- 快速截图入口
- 任务完成提醒

## 5. 状态与动作设计

### 5.1 核心状态表

| 智能体状态 | 小宠物动作 | UI 反馈 |
|---|---|---|
| idle 待命 | 趴着，眼睛半睁，尾巴轻轻卷起 | 菜单栏图标常亮但低调 |
| wake 唤出 | 从输入框边缘探头，眼睛睁大 | 面板弹出，角色轻弹出现 |
| listening 输入中 | 眼睛跟随光标，小爪扶着面板 | 输入框微光 |
| thinking 思考中 | 抱着光球，尾巴慢慢摆动 | 回答区显示流式 loading |
| vision 看图中 | 眼睛发蓝光，凑近截图卡片 | 截图卡片出现扫描光 |
| searching 搜索中 | 低头嗅探文件碎片，尾巴像雷达 | 文件碎片漂浮 |
| tool 执行工具 | 小爪拖拽文件/终端碎片 | 工具调用卡片展开 |
| writing 写入中 | 用尾巴光线串联文本块 | 文件写入状态显示 |
| done 完成 | 跳起，尾巴亮，翼膜展开 | 完成勾选光效 |
| error 出错 | 歪头、眨眼、尾巴打结 | 错误卡片，提供重试 |
| sleep 休眠 | 缩成一团，翼膜盖住身体 | 菜单栏图标变暗 |

### 5.2 第一批必须出图的动作

优先生成 8 个动作资产：

1. **待命**：趴在面板边缘，尾巴卷起。
2. **唤出**：从毛玻璃面板后探头。
3. **观察**：盯着截图卡片，眼睛发亮。
4. **搜索**：在文件碎片之间低头寻找。
5. **思考**：抱着发光线索球。
6. **执行**：用小爪拖动终端/文件卡片。
7. **完成**：跳起，翼膜张开，尾巴发光。
8. **出错**：歪头困惑，尾巴缠住自己。

## 6. 与 kunkun App 的界面结合

### 6.1 悬浮面板

面板设计应该像一个 macOS 原生轻工具，而不是网页。

界面结构：

- 顶部：输入框
- 中部：流式回答
- 侧边/角落：Lumo 状态
- 下方：工具调用时间线
- 右上：截图、附件、历史、设置入口

Lumo 的位置：

- 空状态：居中偏左或输入框边缘探头。
- 对话中：缩小到面板右下角。
- 工具调用：进入工具状态卡片旁边。
- 完成后：跳到回答卡片旁边。

### 6.2 工具调用卡片

工具调用不要只是文字日志，要让 Lumo 参与解释。

示例：

- `读取文件`：Lumo 翻开文件夹。
- `分析图片`：Lumo 眼睛扫过截图。
- `运行命令`：Lumo 拖动发光终端块。
- `写入文档`：Lumo 用尾巴光线把文本串起来。

### 6.3 空状态

空状态文案：

- “把线索丢给我。”
- “我在看着，随时叫我。”
- “今天要抓住哪个灵感？”
- “截图、文件、想法，都可以给我看看。”

## 7. 第一阶段落地范围

第一阶段不要做完整桌宠物理系统，先做“面板内桌宠”。

### 7.1 MVP 功能

- 悬浮面板内显示 Lumo 空状态图。
- 根据后端事件切换 6 个状态：
  - idle
  - thinking
  - vision
  - searching
  - done
  - error
- 工具调用卡片使用对应小图标/小姿态。
- 菜单栏图标用简化眼睛或小脑袋。

### 7.2 暂不做

- 桌面自由爬行
- 碰撞物理
- 复杂骨骼动画
- 语音唤醒同步口型
- 3D 实时模型

原因：MVP 先验证“智能体状态可视化 + IP 陪伴感”，不要过早进入重动画系统。

---

## 8. 二期桌面宠物能力

二期再做真正桌面宠物：

- 可在屏幕边缘停靠
- 可拖动位置
- 可拖拽文件/图片到 Lumo 身上触发分析
- 可双击唤出面板
- 可根据任务状态做小动画
- 可在任务完成后弹出轻提醒
- 可设置“安静模式”
- 可设置是否常驻桌面

## 11. 总结

这套方案的核心不是“给 App 加一个可爱角色”，而是让 Lumo 成为 kunkun 智能体的可视化状态层。

用户不需要看复杂日志，也能通过 Lumo 的眼睛、尾巴、翼膜和动作理解智能体正在做什么。

**kunkun 负责干活，Lumo 负责让用户看见它怎么干活。**
