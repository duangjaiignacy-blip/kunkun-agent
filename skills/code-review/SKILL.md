# 代码审查技能

当困困让你审查代码时，请严格按下面的清单逐条检查，并用中文输出结果。

## 检查清单

1. **正确性**：有没有明显的逻辑错误、边界没处理（空值、除零、越界）？
2. **异常处理**：可能出错的地方有没有 try/except？错误信息是否清晰？
3. **命名**：变量、函数名是否见名知义？有没有 a、b、tmp 这种含糊命名？
4. **安全**：有没有 SQL 注入、命令注入、硬编码密码/密钥、路径遍历风险？
5. **可读性**：函数是否太长（建议拆分超过 50 行的函数）？有没有必要的注释？
6. **重复**：有没有复制粘贴的重复代码，可以提取成函数？

## 输出格式

按以下格式输出，每条问题都要指出**第几行**：

```
## 审查结果

🔴 严重问题
- 第 X 行：xxx（建议：xxx）

🟡 改进建议
- 第 X 行：xxx（建议：xxx）

🟢 总体评价
一句话总结。
```

如果代码没问题，明确说"未发现明显问题"，不要硬凑。
